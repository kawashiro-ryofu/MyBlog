Windows Terminal使用体验

发表于2020-08-14|更新于2021-02-05|软件试用
|阅读量:

Windows Terminal早就不是什么新奇的东西了，在Microsoft Store直接可以找到。有很多新特性。替代Windows原生的Command Prompt再合适不过。我从年初开始，使用至今。

首先，它支持标签页了！

这个X11上的终端早就普及了的功能Windows终于有了！

这是Windows Terminal

这是原生CMD

这是Konsole，KDE的默认终端模拟器

微软终于良心了一次，把终端模拟器该有的功能给补上了[doge]

其次，它原生支持UTF-8

CMD支持UTF-8吗？支持吧。

我使用SSH连接服务器的时候它会自动帮我转换为True Type字体

然后在SSH终端下，你若打开一个使用ncursus库的应用，比如vim、nano等文本编辑器字体会自动还原为CMD的原生字体。看着异常的难受。

在Windows-Terminal下，没有任何烦恼，完美兼容UTF-8字符。

X11的终端：啥？这些不都是我早就有了的吗？

Windows Terminal：可我是微软开发的啊！

X11的终端：

除此之外它还支持WSL的bash终端

不过说实话，CMD下输入bash也可以达到同样的效果。

最后，它支持魔改主题

网上有很多美化教程，这里放一个教程

~我的终端之所以还是黑底白字，是因为我懒得改~~

好的地方说完了，下面说说这款应用的槽点

1.你微软都特意做了个终端模拟器出来了，你不考虑把cmd.exe换掉吗？

我早就习惯Win+R输入CMD调出命令行窗口了。所以即使Windows Terminal是我所想要的终端，但有些时候还是会意外的把原来CMD调出来。

所以，为什么不可以像cmder那样，在用户执行cmd的时候自动调出cmder终端模拟器的窗口，继续使用cmd作为shell。

补充：

Q：如何Win+R打开Windows Terminal

A：Win+R 输入wt

这个我也是在写博文的时候偶然翻到的。

8月20日补充：槽点在于在Win + R 输入 wt 它只是单纯地新建一个窗口而非新建标签页

2.国内输入法有时不灵敏

最近感觉微软拼音的词库少，所以索性换了讯飞输入法。搜狗的广告多。但是在我安装使用的一两天里，讯飞输入法时常失灵。在其它文本框可以输入，偏偏Windows Terminal不行。不知道其他国内输入法有没有这样的问题。

3.非Windows10（包括LTSC这一特殊版本）不能安装

如题，如果第三个条件满足不了的话，出门右转搜索cmder

总结

Windows Terminal是微软官方支持最好的终端模拟器了，即便有些瑕疵，但如果你能适应过来的话，也是不错的。
